# Salesforce Summer Program – Day 2

## Introduction

Salesforce is a cloud-based Customer Relationship Management (CRM) platform that helps organizations manage customer data, automate workflows, improve communication, and develop business applications.

The Salesforce Platform allows developers and administrators to build scalable enterprise applications using configuration tools and programming technologies like Apex and Lightning Web Components.

---

# What is an App in Salesforce?

An App in Salesforce is a collection of tabs, objects, dashboards, reports, and functionalities grouped together for a specific business purpose.

Apps help users access related features in a single workspace.

### Example:
A College Management App may contain:
- Student Records
- Faculty Management
- Attendance Tracking
- Examination Module

---

# What is an Object?

An Object in Salesforce is similar to a database table used to store information.

Each object contains:
- Fields (columns)
- Records (rows)

## Types of Objects

### Standard Objects
Objects already provided by Salesforce.

Examples:
- Account
- Contact
- Opportunity
- Lead

### Custom Objects
Objects created according to business requirements.

Examples:
- Student
- Hospital Patient
- Library Books
- Employee Attendance

---

# What is a Tab?

A Tab is a user interface component that helps users access objects, reports, dashboards, and applications.

Tabs improve navigation and make Salesforce applications user-friendly.

### Examples:
- Student Tab
- Contact Tab
- Opportunity Tab
- Reports Tab

---

# CRM Concepts in Salesforce

Salesforce CRM mainly uses:
- Accounts
- Contacts
- Opportunities

## Account
Represents a company or organization.

## Contact
Represents people associated with accounts.

## Opportunity
Represents potential business deals.

### Example Flow
1. A company is stored as an Account.
2. Employees are stored as Contacts.
3. Sales deals are stored as Opportunities.

This structure helps businesses manage customer relationships effectively.

# Configuration vs Coding

| Configuration | Coding |
|---|---|
| Uses clicks and built-in tools | Uses Apex programming |
| Easy to implement | More flexible |
| Used for simple automation | Used for complex logic |
| Managed by admins | Managed by developers |

## When to Use Configuration
- Validation Rules
- Workflow Automation
- Flow Builder

## When to Use Coding
- Complex Calculations
- External API Integration
- Advanced Business Logic

---

# Multi-Tenant Architecture

Salesforce follows a multi-tenant architecture where multiple organizations share the same infrastructure securely.

## Advantages
- Cost Efficient
- Automatic Updates
- High Scalability
- Better Security
- Easy Maintenance
