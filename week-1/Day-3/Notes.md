# Day 3 Notes – Salesforce Data Modeling

# Objects
Objects are database tables used to store related data.

## Types of Objects
1. Standard Objects
2. Custom Objects

## Examples
- Account
- Contact
- Student
- Course

---

# Fields
Fields store specific information about records.

## Examples
- Name
- Email
- Age
- Department

---

# Records
A record is a single entry inside an object.

## Example
Student Record:
- Name: Rahul
- Age: 20
- Department: CSE

---

# Relationships
Relationships connect objects together.

## Lookup Relationship
Used when related objects can exist independently.

## Examples
- Student linked to Course
- Course linked to Department

---

# Schema Builder
Schema Builder is a visual tool used to create and understand relationships between objects.

## Benefits
- Easy visualization
- Better understanding
- Simplifies data modeling

---

# Formula Fields
Formula Fields automatically calculate values.

## Examples
- Full Name
- Percentage
- Remaining Seats

## Benefits
- Reduces manual work
- Prevents calculation errors
- Updates automatically

---

# Validation Rules
Validation Rules prevent incorrect data from being saved.

## Examples
- Email cannot be blank
- Age cannot be negative
- Seats cannot exceed limit

## Benefits
- Improves data quality
- Maintains consistency
- Prevents invalid records

---

# Business Logic Without Coding
Salesforce supports no-code automation using:
- Formula Fields
- Validation Rules
- Flow Builder
- Process Builder

---

# Key Learnings
- Enterprise systems need structured data.
- Relationships are important.
- Automation improves efficiency.
