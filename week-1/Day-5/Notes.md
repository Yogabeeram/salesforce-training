
# Day 5 Notes – Salesforce Apex Introduction

# What is Apex?

Apex is Salesforce’s programming language used to build custom business logic and automation.

It is:
- Object-oriented
- Similar to Java
- Cloud-based
- Integrated with Salesforce database

---

# Why Apex Exists

Salesforce no-code tools are powerful but limited for:
- Complex processing
- Advanced calculations
- External integrations
- Large-scale automation

Apex solves these limitations.

---

# Important Apex Concepts

## Variables
Store data values.

Example:
- Student Name
- Fee Amount

---

## Conditional Logic

Used for decision-making.

Example:
IF attendance < 75%
THEN send notification

---

## Loops

Used to process multiple records.

Example:
Loop through all students with pending fees.

---

## Classes

Apex classes contain reusable logic and methods.

---

# Database Operations

## DML Operations

Used to:
- Insert records
- Update records
- Delete records

---

# SOQL Basics

SOQL = Salesforce Object Query Language

Used to retrieve Salesforce data.

Example:
Retrieve all students with pending fees.

---

# Declarative vs Programmatic Development

## Declarative Development
Uses:
- Flows
- Validation Rules
- Configuration

Advantages:
- Faster
- Easier
- No coding required

---

## Programmatic Development
Uses:
- Apex
- Triggers
- Custom logic

Advantages:
- Flexible
- Powerful
- Advanced functionality

---

# Flow vs Apex

| Flow | Apex |
|---|---|
| No-code | Programming |
| Simple automation | Complex automation |
| Easy maintenance | High flexibility |

---

# Real Examples Where Apex Is Needed

1. Payment gateway integration
2. Complex eligibility calculation
3. Multi-level approval logic

---

# Business Understanding

Enterprise systems require:
- Automation
- Validations
- Integrations
- Custom business rules

Salesforce combines:
- Declarative tools
- Apex programming

to build scalable enterprise applications.

---

# Important Learning

Developers should always:
- Prefer no-code first
- Use Apex only when needed
- Avoid unnecessary coding
- Focus on business requirements
